Architecture Overview

Inside PMIS

A layered memory system that structures knowledge into a weighted, temporal, evidence-driven graph. Here's how the pieces fit together.

Data Model
Three-level knowledge graph.
All knowledge is organized in a strict hierarchy. Broad domains contain skill areas, which contain atomic insights. Every node carries a weight that evolves based on real-world outcomes.
Domain
The broadest category of work.
Reusable across many conversations. Carries a quality score derived from the average outcome of all tasks within it. Examples: "B2B Outreach", "Security Platform", "Retail AI".
Skill Area
A distinct phase or competency within a domain.
Groups related insights by skill. Weight is computed as the mean of its children's evidence-weighted scores. Examples: "Email Copywriting", "Target Research", "Pricing Strategy".
Insight
One atomic, reusable piece of knowledge.
The fundamental unit. Each insight has a weight (0–1) that begins as an estimate and converges toward evidence over time. Tagged for multi-signal retrieval. Carries temporal metadata: when it was used, how often, how consistently.

The hierarchy is self-deduplicating. Storing knowledge about "B2B Outreach" fifty times doesn't create fifty trees — it merges into one, growing richer with each contribution. A fuzzy matching engine recognizes when titles refer to the same concept, even with different phrasing.

Retrieval
Triple-fusion search pipeline.
Retrieval runs three independent search strategies in parallel, then merges their rankings into a single result set. This is why PMIS finds relevant knowledge even when queries use different vocabulary than what was stored.
Tag Engine
Matches domain-aware tags generated at store time. Fast set-overlap on structured vocabulary.
structured matching
Keyword Engine
Full-text search with term importance weighting. Expands queries with related terms from the knowledge graph.
relevance scoring
Semantic Engine
Computes document-level similarity using term frequency patterns. Catches conceptual matches that keywords miss.
conceptual matching

Post-Merge
Quality, temporal, and context-aware re-ranking.
After fusion, results are further adjusted by: domain quality score (how well has this domain performed historically?), temporal stage (is this knowledge current or stale?), and mode affinity (does this domain match the type of work being done?).
Output
Ranked context pack with provenance signals.
Each returned insight carries: its evidence-based weight, temporal stage, whether it appeared in the highest-scoring historical task, and if execution details matter. The caller knows not just what to use, but how much to trust it.
Signal Systems
Four independent signals shape every retrieval.
Unlike flat memory stores, PMIS weighs knowledge on multiple axes. Each signal adds a dimension of intelligence that simple keyword systems can't replicate.

Knowledge matures through four stages.
