P9+ pipeline: 10 steps from query to result
1
Query expansion
Append last 2 conversation turns (query text only, not SC titles). Prevents retrieval feedback loops.
Current query weighted 60% in BM25
2
Tag search
Jaccard overlap between query tokens + prior anchor tags and each anchor's tag set.
Weight: 40% at current scale (Small (<500))
3
BM25 search
Current query (60%) + expanded (40%). IDF weighting auto-resists vocabulary noise at scale.
Weight: 35% at current scale
4
Vector search
TF-IDF cosine similarity on path-enriched text. Requires sklearn (optional).
Weight: 25% at current scale. Status: Inactive (install sklearn)
5
Triple merge
Phase A: Weighted raw blend (tag/bm25/vec). Phase B: Reciprocal Rank Fusion across 3 lists. Combined 70% raw + 30% RRF.
6
Temporal boost
Memory stage multiplier: established 1.5x, active 1.2x, impulse 0.8x, fading 0.5x
7
Mode boost
6D personality match: 0.5 + cosine(query_mode, sc_mode). Range 0.5x to 1.5x
8
Tree-level quality
Evidence-based anchor weights (blend initial + feedback). Best context tree drives SC bonus. Winning structure anchors get +0.2 sort priority.
9
Transfer edges
Cross-domain links: if SC-A in results, add linked SC-B at 40% score. 0 transfer edges active.
10
Return top 3
Full SC → Context → Anchor trees. Contexts sorted by evidence quality. Winning anchors first.
